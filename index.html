Linux Basics

Fundamental concepts every Linux administrator must know before diving into commands.

💡 What is Linux? Linux is a free, open-source, Unix-like operating system kernel created by Linus Torvalds in 1991. It powers servers, supercomputers, mobile devices (Android), and embedded systems worldwide.

🖥️ Essential First Commands

uname
Display system information
Beginner

Syntax

$ uname [options]

Common Options

OptionDescription
-aAll information (kernel, hostname, OS, architecture)
-rKernel release version
-nNetwork node hostname
-mMachine hardware architecture
-sKernel name
-oOperating system name

Examples

  • uname -aShow all system information
  • uname -rShow only kernel version: 6.8.0-generic
whoami / id
Display current user and group info
Beginner

Examples

  • whoamiDisplay current username
  • idShow user ID, group ID, and all groups
  • id usernameShow info for a specific user
  • whoShow who is logged on
  • wShow who is logged on and what they're doing
pwd
Print Working Directory — show current location
Beginner

Examples

  • pwdDisplay current directory: /home/admin
  • pwd -PPhysical path (resolve symlinks)
cd
Change Directory — navigate the filesystem
Beginner

Examples

  • cd /path/to/dirNavigate to absolute path
  • cd dirnameNavigate to relative path (subdirectory)
  • cd ..Go up one directory level
  • cd ../..Go up two directory levels
  • cd ~Go to home directory
  • cd -Go to previous directory
  • cd /Go to root directory
ls
List directory contents
Beginner

Common Options

OptionDescription
-lLong format (permissions, owner, size, date)
-aShow hidden files (starting with .)
-hHuman-readable file sizes
-tSort by modification time (newest first)
-rReverse order
-RRecursive listing (subdirectories)
-laLong format + hidden files (most common)

Examples

  • ls -lahDetailed listing with human-readable sizes
  • ls -ltSort by time, newest first
  • ls *.confList only .conf files
  • ls -lR /etcRecursive listing of /etc
  • ls -lhSSort by size, largest first
man / info / help
Access command documentation
Beginner

Examples

  • man lsOpen manual page for ls
  • man 5 passwdOpen section 5 (file formats) for passwd
  • info coreutilsInfo documentation for coreutils
  • ls --helpQuick help for ls command
  • apropos "search"Search man pages by keyword
  • whatis commandOne-line description of command
history
View command history
Beginner

Examples

  • historyShow all previous commands
  • history 20Show last 20 commands
  • !numberRe-run command by history number
  • !!Re-run last command
  • !stringRe-run last command starting with "string"
  • Ctrl+RInteractive reverse search through history
  • history -cClear command history
clear / reset
Clear terminal screen
Beginner
  • clearClear the terminal screen
  • resetReset terminal (fixes display issues)
  • Ctrl+LKeyboard shortcut to clear screen

Linux Filesystem Hierarchy

Understanding the Linux directory structure is fundamental to system administration.

DirectoryPurpose
/Root directory — the top of the filesystem hierarchy
/binEssential user binaries (ls, cp, cat, etc.)
/sbinSystem binaries (fsck, init, iptables)
/etcSystem configuration files
/homeUser home directories
/rootRoot user's home directory
/varVariable data (logs, mail, spool, cache)
/tmpTemporary files (cleared on reboot)
/usrUser programs and data (secondary hierarchy)
/optOptional/add-on software packages
/devDevice files (hardware interfaces)
/procVirtual filesystem for process/kernel info
/sysVirtual filesystem for system hardware info
/bootBoot loader files (kernel, initrd, grub)
/libEssential shared libraries and kernel modules
/mediaMount points for removable media
/mntMount points for temporary filesystems
/srvService data (web, FTP, etc.)
/runRuntime variable data (PIDs, sockets)
💡 Tip: FHS The Filesystem Hierarchy Standard (FHS) defines the directory structure. You can read more with: man hier
